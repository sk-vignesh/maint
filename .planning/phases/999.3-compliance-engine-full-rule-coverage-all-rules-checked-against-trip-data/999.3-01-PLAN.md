### Task 2: Fix the hasDriving Guard Bug in `assimilated.ts` (D-10)

<read_first>
- `apps/v4/lib/compliance-engine/assimilated.ts` (full file — focus on L130–220, the per-day rules loop)
</read_first>

<action>
In `apps/v4/lib/compliance-engine/assimilated.ts`, locate the per-day rules loop (approximately L130–220).

**Currently the loop starts with:**
```typescript
// ── Per-Day Rules ─────────────────────────────────────────────────────────
for (const day of days) {
  if (day.isRestDay) continue
  if (!day.hasDriving) continue  // ← THIS IS THE BUG
  // ... driving-specific checks (daily drive limit, breaks, etc.)
```

**Replace the inner `if (!day.hasDriving) continue` and everything following in the loop** so that:
1. Non-driving working days get `EU_DAILY_WORK_LIMIT` checked (13h warn / 15h violation) then `continue`
2. Only driving days proceed to driving-specific checks (L135 onwards)

The new loop structure must be:

```typescript
// ── Per-Day Rules ─────────────────────────────────────────────────────────
// Constants for daily working hours limit (EU art. 6 interpretation for duty)
const MAX_DUTY_WARN_MINS    = 13 * 60   // 780 min — warn when approaching
const MAX_DUTY_REDUCED_MINS = 15 * 60   // 900 min — violation (reduced rest day)

for (const day of days) {
  if (day.isRestDay) continue

  // ── Non-driving working day: only check daily working hours limit ──────
  // (Break compliance is assumed for NON_DRIVING_DUTY — self-managed by driver)
  if (!day.hasDriving) {
    const dutyMins = day.totalDutyMinutes
    if (dutyMins > MAX_DUTY_REDUCED_MINS) {
      issues.push({
        ruleId:     "EU_DAILY_WORK_LIMIT",
        severity:   "violation",
        date:       day.date,
        driverUuid: day.driverUuid,
        message:    `Daily working hours exceeded — ${Math.round(dutyMins / 60 * 10) / 10}h on duty (15h absolute maximum)`,
        calculation: `${dutyMins}min duty > ${MAX_DUTY_REDUCED_MINS}min limit`,
        ruleset:    "ASSIMILATED",
      })
    } else if (dutyMins > MAX_DUTY_WARN_MINS) {
      issues.push({
        ruleId:     "EU_DAILY_WORK_LIMIT",
        severity:   "warning",
        date:       day.date,
        driverUuid: day.driverUuid,
        message:    `Daily working hours approaching limit — ${Math.round(dutyMins / 60 * 10) / 10}h on duty (13h standard / 15h with reduced rest)`,
        calculation: `${dutyMins}min duty > ${MAX_DUTY_WARN_MINS}min warn threshold`,
        ruleset:    "ASSIMILATED",
      })
    }
    continue  // no driving-specific checks for this day
  }

  // ── From here: only DRIVING days ────────────────────────────────────────
  // (existing code for daily driving limit, fortnightly, weekly, breaks etc.)
```

**Important:** Place the `MAX_DUTY_WARN_MINS` and `MAX_DUTY_REDUCED_MINS` constants BEFORE the `for (const day of days)` loop, not inside it. If there is already a constants section at the top of the function, add them there.

After this change:
- Rest days → skip (unchanged)
- NON_DRIVING_DUTY days → check 13h/15h limit, then continue
- DRIVING days → all existing checks (unchanged)
</action>
